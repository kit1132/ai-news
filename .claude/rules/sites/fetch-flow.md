# ソース取得フロー

## 取得手順

各ソースの取得は以下の順序で試行する。

1. **プライマリURLをWebFetchで取得**
2. 403 / 429 / タイムアウトの場合 → **代替URLがあればWebFetchで試行**
3. それでも失敗した場合 → **WebSearchで `site:ドメイン 検索キーワード 当年` を実行**
4. `site:` 検索で結果0件の場合 → **`site:` を外して `検索キーワード 当年` で再検索**
5. すべて失敗した場合 → ダイジェストに「取得不可（手動確認推奨）」と記載

## フォールバック発生時の記録ルール

フォールバック（手順2以降）を使用した場合、ダイジェスト末尾の **「改善メモ」セクションに以下を記録する**：

```
- [ソース名] 取得失敗
  - 失敗URL: <URL>
  - ステータスコード: <403/429/timeout など>
  - 成功した方法: <代替URL WebFetch / WebSearch site: / WebSearch フリーワード>
  - 日時: <YYYY-MM-DD>
```

同じURLの失敗が2回以上記録されたら、`daily-sources.md` の取得方法欄を更新することを強く推奨。

## WebSearch利用時の注意事項

- `site:` オペレータは非対応の場合がある。結果0件なら必ず `site:` を外して再検索する
- WebSearchに日付フィルタ機能はない。クエリに年（例: `2026`）を含めて新しい結果を優先する
- インデックス反映に数時間〜数日のラグがある。速報性が必要な情報には限界がある
- WebSearch結果はURLとスニペットのみの場合がある。必要に応じて結果内のURLをWebFetchで追加取得する
